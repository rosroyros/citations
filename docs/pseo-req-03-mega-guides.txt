## 1. MEGA GUIDE OVERVIEW

### 1.1 Purpose
Mega guides are comprehensive, authoritative resources (5,000-8,000 words) that serve as:
- Hub pages for topical clusters
- Entry points from organic search
- Link attractors for SEO
- Complete references users bookmark

### 1.2 Characteristics
- **Length:** 5,000-8,000 words
- **Reading Time:** 20-30 minutes
- **Examples:** 10-15 minimum
- **Internal Links:** 20-30 to related pages
- **Update Frequency:** Quarterly review
- **Priority:** Phase 1 (create first)

#### **VALIDATION CHECKLIST**

```markdown
## Complete Validation Checklist

Use this checklist to systematically check your citations:

### Reference List Format

- [ ] Page title "References" (bold, centered)
- [ ] Double-spaced throughout
- [ ] Hanging indent (0.5 inches) for each entry
- [ ] Alphabetical order by first author's last name
- [ ] No extra line breaks between entries

### Individual Citation Elements

- [ ] Author names: Last, F. M. format
- [ ] Ampersand (&) before last author
- [ ] Year in parentheses
- [ ] Article titles in sentence case
- [ ] Article titles NOT italicized
- [ ] Journal names in title case
- [ ] Journal names italicized
- [ ] Volume number italicized
- [ ] Issue number in parentheses, not italicized
- [ ] Page numbers without "pp." prefix
- [ ] DOI as https://doi.org/ format
- [ ] No period after DOI

### Cross-Checking

- [ ] Every in-text citation has matching reference
- [ ] Every reference is cited in text
- [ ] Years match between in-text and reference list
- [ ] Spelling consistent between in-text and reference

### Special Elements

- [ ] Et al. used correctly (3+ authors)
- [ ] Multiple works by same author have a, b, c suffixes
- [ ] Retrieval dates only for dynamic content

[Download Printable Checklist]

[Run Automated Check]
```

#### **APA 6 VS 7 COMPARISON**

```markdown
## APA 6th vs 7th Edition: What Changed

If you're transitioning from APA 6 to APA 7, here are the key changes:

| Element | APA 6 | APA 7 | Why It Changed |
|---------|-------|-------|----------------|
| Issue number | Optional if continuous pagination | Always required | Aids retrieval |
| Page numbers | pp. 45-67 | 45-67 | Simplification |
| DOI format | doi:10.xxxx | https://doi.org/10.xxxx | Web standards |
| Author limit | List up to 7, then et al. | List up to 20 | More inclusive |
| Running head | Required on all pages | Not required for students | Simplification |
| Location | City, State: Publisher | Just Publisher | Outdated info |

**Most Common Transition Errors:**
1. [Error from using old format]
2. [Another common mistake]
3. [Third mistake]

**Quick Conversion Guide:**
- If you have "pp." before pages → Remove it
- If DOI starts with "doi:" → Change to "https://doi.org/"
- If issue number missing → Add it (always required now)
- If location before publisher → Remove location

[See complete APA 7 changes guide]
```

## 4. CONTENT GENERATION WORKFLOW

### 4.1 Creating a New Mega Guide

**Step 1: Research and Outline**
- Define primary keywords
- Identify target audience
- List main sections (8-12 H2s)
- Identify unique elements for this guide
- Gather 15+ relevant examples

**Step 2: Content Development**
- Write hero section (H1, subtitle, summary)
- Write TL;DR box
- Write each H2 section (400-800 words each)
- Create examples section (10-15 examples)
- Create error section (10-15 errors)
- Build validation checklist
- Write FAQ (8-12 questions)
- Add related resources

**Step 3: Enhancement**
- Add internal links (20-30 throughout)
- Insert CTAs (5 placements)
- Add visual elements (tables, lists)
- Format examples with proper styling
- Add metadata (reading time, update date)

**Step 4: Quality Assurance**
- Verify all rules against APA manual
- Check example authenticity
- Test all internal links
- Verify CTA functionality
- Proofread for tone consistency
- Validate reading level
- Check for completeness

**Step 5: SEO Optimization**
- Optimize title tag
- Write meta description
- Add schema markup
- Optimize images (if any)
- Check keyword density
- Verify header hierarchy

## 5. MEGA GUIDE STANDARDS

### 5.1 Writing Standards

**Voice and Tone:**
- Second person ("you") addressing reader directly
- Active voice 80%+ of sentences
- Conversational yet authoritative
- Empathetic to citation struggles
- Solution-focused

**Structure:**
- Clear hierarchy (H1 → H2 → H3)
- Scannable (lists, tables, bold text)
- Logical flow between sections
- Smooth transitions
- Consistent formatting

**Quality Metrics:**
- Flesch Reading Ease: 60-70 (standard)
- Flesch-Kincaid Grade: 10-12
- Passive voice: <20%
- Sentence length: 15-20 words average
- Paragraph length: 3-5 sentences average

### 5.2 Example Standards

**All Examples Must:**
- Use real sources (from Google Scholar, PubMed, etc.)
- Include publication dates 2020-2024
- Show complete citation (not truncated)
- Include both reference and in-text formats
- Annotate key formatting elements
- Be properly formatted according to APA 7

**Example Format:**
```markdown
**Example [#]: [Description]**

**Reference list format:**
[Complete citation with proper formatting]

**In-text citation (parenthetical):** (Author, Year)
**In-text citation (narrative):** Author (Year)

**Key formatting notes:**
- [Element 1] is formatted as [description]
- [Element 2] appears [location] and is [formatted how]
- Note that [special consideration]

**Common error:** [What people often get wrong]
```

### 5.3 Internal Linking Standards

**Required Links:**
- Link to related source type pages (5-10)
- Link to related error pages (5-10)
- Link to validation guides (3-5)
- Link to other mega guides (2-4)
- Link to checker tool (5 times)

**Linking Best Practices:**
- Natural anchor text (not "click here")
- Contextually relevant placement
- Open in same tab (internal links)
- No broken links
- Balanced distribution throughout content

### 5.4 Update Requirements

**Quarterly Review:**
- Check for APA updates or clarifications
- Update examples with more recent publications
- Review and update statistics
- Check competitor guides for gaps
- Update related links

**Update Indicators:**
- "Last updated" date in hero section
- Change log (internal tracking)
- Version number (internal)
