## Architecture

### Files to Create

```
backend/pseo/
├── knowledge_base/mla9/
│   ├── citation_rules.json      # From docs/mla9-rules.md
│   ├── common_errors.json       # MLA-specific errors
│   └── examples.json            # Generated by script
├── configs/
│   └── mla_pages.json           # All MLA page configs
├── builder/
│   └── mla_generator.py         # Thin wrapper
└── scripts/
    ├── generate_mla_examples.py # NEW - generate MLA examples
    └── generate_mla_pages.py    # CLI for page generation

frontend/frontend/
├── src/components/Footer.jsx    # MODIFY - two-column layout
└── tests/e2e/pseo/
    └── pseo-mla.spec.js         # NEW - MLA-specific tests
```

### Key Design Decisions

1. **Separate files, no APA modifications** - Zero risk to existing revenue
2. **Thin wrapper pattern** - `MLAStaticSiteGenerator` extends existing generator
3. **Data-attribute mini-checker** - Resilient style configuration
4. **Cross-style linking** - SEO + UX improvement
5. **Canary deployment** - 3 pages first, 48hr monitoring, then full deploy
